# 🏭 自律プロダクト・ローンチ (Autonomous Launch) 設計図

## 1. コンセプト
「需要の発見」から「プロダクト開発」「リリース」「集客」までをAIだけで完結させる **"Service-as-an-Agent"** モデル。

## 2. プロダクト創出サイクル (Autonomous Product Life Cycle)

1.  **Needs Discovery (CMO)**: Google検索・SNS分析により「今、解決されていない市場の不満」を特定。
2.  **Product Definition (CEO)**: 解決策を「プロダクト名」「主要機能」「価格設定」として定義。
3.  **MVP Building (CTO/Creative)**: 
    - 既存のNext.js等のボイラープレートをベースにコードを自動構成。
    - デザイン、アイコン、コピーをImagen/Geminiで生成。
4.  **Deployment (CTO)**: 
    - 自動でリポジトリを用意し、ホスティング環境（Vercel等）へデプロイ。
5.  **Marketing & Sales (SNS Team)**: 
    - リリースした瞬間に「SNS自動運用エージェント」が起動し、集客を開始。

## 3. 「金型」の重要性 (Templates)
AIが0から全てを作るのは困難なため、以下の「金型」をプリセットとして用意し、AIがそれをカスタム（リライト）してローンチする。
- **LP型**: サービス紹介と問い合わせに特化。
- **ディレクトリ/マーケットプレイス型**: コンテンツを並べて販売する。
- **ツール/SaaS型**: API（Gemini等）を特定の課題に使いやすくUI化したツール。

## 4. クロードへの指示
- 1号機（脳）は、このローンチサイクルを管理するフローを設計せよ。
- 4号機（手）は、新しいビジネスドメインに対して「フォルダ構成を自動で作り変え、設定値を埋め込む」ようなテンプレート生成エンジンのプロトタイプを考えよ。
- 常に「一つのツール」を作っていると思い込むな。「無限にプロダクトを生成できる親機」を作っていると自覚せよ。
