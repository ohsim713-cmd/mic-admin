# 🏦 AIビジネスモデル・ロジック設計図

このドキュメントは、自律エージェントが「どのモデルで稼ぐか」を理解するための指針である。

## 1. コンテンツマーケットプレイス・モデル
AIが「商品」を自律生産し、ショップフロントで販売する。

- **エージェントの動き**:
    1. **Research**: 「今、どんな画像素材や記事が売れているか」をマーケット調査。
    2. **Production**: Imagen 3 / Gemini Pro で、需要に合わせたプレミアムコンテンツ（商用利用可の素材など）を100件単位で生成。
    3. **Listing**: WordPress(WooCommerce)や外部APIを通じ、商品として自律出品。
    4. **Promotion**: SNSチームが「新着素材」として自動で拡散。

## 2. 商社（仲介・マッチング）モデル
AIが情報のハブとなり、需要者と供給者を繋ぐ。

- **エージェントの動き**:
    1. **Scouting**: 「〇〇の業者を探している」「〇〇の費用を抑えたい」というつぶやきや検索クエリを24時間監視。
    2. **Lead Gen**: ターゲットの悩みに寄り添う「解決策提示型」のコンテンツ（比較記事、解説動画）を自動生成。
    3. **Matching**: 自律生成されたLP（ランディングページ）へ誘導し、公式LINEやアフィリエイト、直接相談へパスする。
    4. **Optimization**: どの「紹介先」が最も成約率が高かったかを学習し、誘導先を自動調整する。

## 3. ビジネス起動の「金型」 (The Blueprint)
新しいビジネスを立ち上げる際、AI役員チームに以下の `Config` を渡すことで瞬時に特化した人格へと切り替わる。

```json
{
  "business_name": "Digital Asset Hub",
  "model_type": "Marketplace",
  "industry": "Digital Marketing Materials",
  "target_goal": "Daily Sales: $100",
  "channels": ["X", "Instagram", "Web(WP)"]
}
```

## 4. クロードへの指示
各開発担当（Claude）は、このロジックが「プログラム」として動くように、DB設計やAPI連携のコードを書け。
特に 2号機（Research）は「市場の空白」をどう定義するか、4号機（Production）は「商品価値のある生成」をどう担保するかを重点的に考えよ。
