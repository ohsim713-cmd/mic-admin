# /pdca - 自動改善サイクルスキル

成功パターンを分析し、投稿品質を改善します。

## 使い方

```
/pdca [フェーズ]
```

**例:**
- `/pdca` → 全フェーズ実行
- `/pdca check` → 分析のみ
- `/pdca act` → 改善提案のみ

## 実行手順

### Plan（計画）

1. **現状分析**
   ```bash
   curl -s "http://localhost:3000/api/dm-hunter/learning-stats"
   ```

2. **パフォーマンス取得**
   ```bash
   curl -s "http://localhost:3000/api/dm-hunter/performance-analyzer"
   ```

### Do（実行）

- 通常の投稿生成フローで自動適用

### Check（確認）

1. **成功パターン確認**
   - frontend/data/success_patterns.json を読み込み
   - 高スコアパターンを抽出

2. **分析レポート**
   ```
   【成功パターン分析】

   ✅ 高スコア要因
   - 「ぶっちゃけ」で始まる投稿: 平均8.5点
   - 具体的数字（月○万円）含む: 平均8.3点
   - 3段落構成: 平均8.2点

   ❌ 低スコア要因
   - 長文（4段落以上）: 平均6.8点
   - CTA弱い: 平均7.1点
   ```

### Act（改善）

1. **改善提案生成**
   - どのパターンを強化すべきか
   - どのパターンを避けるべきか

2. **ナレッジベース更新提案**
   - knowledge/*.json の更新候補

## 品質基準

| 項目 | 配点 | 説明 |
|------|------|------|
| 共感 | 0-3点 | 冒頭フレーズの共感度 |
| メリット | 0-2点 | 具体的な数字 |
| CTA | 0-2点 | DM誘導の強さ |
| 信頼性 | 0-2点 | 第三者実績 |
| 緊急性 | 0-1点 | 時間制限 |
